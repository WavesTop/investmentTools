InvestInsight 是一个基于 Qt/C++ 开发的桌面级智能投资分析应用，面向 A 股市场板块级投资决策。系统从新浪财经、腾讯财经、东方财富等多个数据源获取实时行情、K线数据、资金流向和财经新闻，结合技术分析、量化策略回测、周期分析及可选的 AI 深度分析能力，为用户提供完整的板块投资分析报告与操作建议。

🏗️
技术栈
C++17 + Qt 5/6（Core, Widgets, Network, Concurrent），CMake 构建，macOS/Linux 平台
📡
数据来源

📈
分析能力
MACD/RSI/KDJ/MA/BOLL/ATR 技术指标 + 5 种策略回测 + 周期分析 + 板块轮动检测
🤖

评分公式（FormulaBreakdown）

每个板块的综合评分由以下因子加权得到，最终映射为 AdviceAction（增配/持有/减仓）：

因子来源说明
techFactorTechIndicators基于 MACD/RSI/KDJ/MA 的技术动量综合评分
newsSentimentSignalExtractor新闻情感均值（正面→看多，负面→看空）
trendMomentumK线数据5日/20日动量因子
flowFactor资金流主力资金净流入方向与强度
meanReversionPenaltyK线偏离度均值回归惩罚（过度偏离MA时减分）
valuationFactor估值百分位当前价格在历史区间中的位置
crowdingFactor成交/换手/动量拥挤度修正（过度拥挤时扣分）
dataQualityWeight数据覆盖率数据完整性权重（缺失越多权重越低）
sourceConsistencyWeight来源一致性多数据源一致性加权

8核心功能特性

  • 多源数据融合 — 同时接入新浪、腾讯、东方财富、Yahoo等多个数据源，通过名称归一化（去除"概念"/"板块"/"行业"后缀）和模糊匹配合并数据，确保覆盖面和容错性
  • 投资组合管理 — 支持多板块、多批次持仓配置；跟踪首次收益日起的累计收益；智能计算今日/累计盈亏；已卖出标记自动过滤
  • 累计收益计算 — 从用户配置的首次收益日（firstEarnDate）开始，在 ETF 代理 K 线中找到对应的起始价格，计算至今的累计收益率。未来日期自动归零，K线缺失时使用最后一根K线作为保守回退
  • 多因子评分体系 — 综合技术面（MACD/RSI/KDJ等）、消息面（新闻情感）、资金面（主力流向）、估值面（百分位）、动量因子，加上数据质量和来源一致性权重，输出 -1 到 +1 的标准化评分
  • 5 种策略回测 — MACD金叉策略、RSI超卖反弹、均线多头突破、放量突破策略、布林带回归策略 — 独立统计胜率/平均收益/最大回撤，并基于最新K线重新评估信号（非历史最后一笔）
  • ATR(14) 动态止盈止损 — 根据每个板块14日平均真实波幅独立计算，高波动板块止损区间更大、止盈目标更高，与主策略评分保持一致
  • 周期分析 + 趋势判断 — 峰谷检测识别板块周期（底部蓄力/复苏上行/扩张加速/顶部滞涨/收缩回调），数据不足时回退到 MA5/MA20/MA60 趋势判断（上升趋势/下降趋势/横盘整理/低位震荡/高位震荡）
  • 板块轮动检测 — 跨板块动量排名、轮动信号识别、六维市场风险雷达图数据
  • 资金流向分析 — 新浪概念+行业双维度资金流获取；历史资金流缓存（QSettings持久化60天）；未匹配板块使用量价估算 fallback
  • 多源新闻聚合 — 10+ 来源并行抓取，关键词→板块映射，情感极性分析（看多/看空/中性），新闻按时间倒序展示并附带可点击原文链接
  • 图表系统 — QPainter 自定义绘制：3 组趋势图（3月/6月/1年折线+面积填充）、3 组 K 线图（日K/周K/月K 真实 OHLC 蜡烛图）、成交量柱状图、MACD 柱状图、资金流面板 — 每张图底部4个等间距日期标注
  • 休市智能检测 — 综合星期、时间（9:25后判定交易时段）、最新K线日期判断市场是否休市，UI动态切换"今日"/"上一交易日"标签
  • AI 深度分析（可选） — 支持 DeepSeek / OpenAI / 通义千问 / Claude / Gemini 等多提供商；Stage 1 新闻摘要 → Stage 2 Top-N 板块深度分析 → 对话式查询
  • 自动验证模式 — --auto-analyze 命令行参数支持无人值守分析运行，用于日志输出验证计算正确性

currentSignal 判定逻辑

信号基于最新K线的实时状态评估，而非历史最后一笔交易结果：

信号判定条件
持仓中回测结束时策略仍处于持仓状态（entryPrice > 0）
买入信号最新一根K线触发买入条件
卖出信号最新一根K线触发卖出条件
近期买入前一根K线触发买入条件（最新K线已过，但信号仍近期有效）
观望最新两根K线均无明确信号

止盈百分比+ATR(14)% × 2.5，结合阻力位距离[+2%, +20%]

14关键问题修复记录

API 数据源迁移
东方财富 push2.eastmoney.com 持续返回 RemoteHostClosedError，全面切换到新浪/腾讯 API 作为主力数据源，东方财富降级为备用
Gzip 解压问题
HttpClient 显式设置 Accept-Encoding: gzip 导致 Qt 无法自动解压响应，移除该头部恢复正常
GBK 编码处理
新浪资金流 API 返回 GBK 编码 JSON，UTF-8 解析失败，增加 getGbk() 回退机制
累计收益计算修复
firstEarnDate 无对应K线时，从使用第一根K线（导致虚假高收益）改为使用最后一根K线（正确归零）
止盈止损差异化
所有板块止盈止损相同（固定 -3%/+3%），引入 ATR(14) 基础的动态计算，各板块根据波动率独立设置
休市判断智能化
仅靠 lastDataDate != today 判断导致交易日误判为休市，改为综合星期+时间+K线日期的多条件判断
资金流覆盖扩展
仅获取概念板块(fenlei=1)资金流，增加行业板块(fenlei=0)获取，匹配率从 ~180 提升到 ~228 个板块
策略回测信号修复
currentSignal 取历史最后一笔交易结果（几乎全是"卖出"），改为基于最新K线重新评估策略条件
持仓建议金额单位
用户输入"元"但显示文本直接标注"万元"，改为根据金额大小自动切换单位（≥10000 → 万元，否则 → 元）
图表系统重构
移除事件标注，新增 3组趋势图(3月/6月/1年) + 3组K线图(日K/周K/月K)，K线数据从 120 扩展到 250 根
板块详情页布局重构
将投资结论移至页面最顶部（操作建议+预期收益/风险+展望），图表区次之，关键事件与新闻排最后
新闻可点击链接
新增 NewsEntry 结构体存储新闻 URL，板块详情页新闻列表支持点击直接跳转原文
