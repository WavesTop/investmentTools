板块详情页

每个板块独立经历以上全部计算步骤，最终生成 SectorSnapshot（约 60+ 字段），渲染为用户看到的板块详情页面。

2阶段一：多源数据拉取（并行）

三路并行拉取，总耗时取决于最慢的一路（通常 15-30 秒）：

路线 A：板块数据
SectorFetcher
板块列表（新浪概念~175 + 行业~48）→ ETF 代理日 K 线（250根）→ 资金流（概念+行业双维度）→ 估值百分位 → 拥挤度
路线 B：新闻数据
NewsProvider
10+ 来源并行（东财/新浪/同花顺/华尔街见闻/财联社等）→ 关键词→板块映射 → 去重与质量评分
路线 C：大盘环境
MarketContext
上证/深证/创业板（腾讯实时）→ 美股三大指数（Yahoo）→ 涨跌家数 → 北向资金代理 → 风险情绪

关键计算：估值百分位

// 将 250 根 K 线的收盘价排序，看当前价格在历史中的位置 historicalCloses = sort(dailyBars[0..n].close) rank = count(close < currentPrice) pePercentile = rank / n × 100 // 0 = 极低估, 100 = 极高估

关键计算：拥挤度指数

turnoverRatio = 近5日均换手 / 历史均换手 volPercentile = 当前成交量在历史中的百分位 momFactor = min(|5日涨幅| / 10, 1.0) crowdingIndex = min(turnoverRatio/2, 1) × 40 + volPercentile × 0.35 + momFactor × 25 // 范围 [0, 100]，>70 为拥挤

关键计算：资金流信号

// 5日平均净流入（亿元），通过 tanh 压缩到 [-1, +1] flow5dAvg = mean(最近5日净流入) flowSignal = tanh(flow5dAvg / 30) // 30亿为归一化基准

关键计算：热度评分

hotScore = |changePct| × 0.35 + min(turnoverRate, 30)/30 × 100 × 0.35 + min(涨跌家数, 200)/200 × 30

3阶段二：信号提取与情感聚合

流程

  1. 关键词信号提取（SignalExtractor）：从 RawInsight 新闻标题/正文中提取关键词，匹配板块，赋予方向（Positive/Negative）和强度值
  2. AI 新闻摘要（可选 Stage 1）：将新闻标题发送给 LLM，返回结构化的板块信号（sentiment 值），与关键词信号合并
  3. 按板块聚合：每个板块累加所有信号的 sentiment 值，统计正面/负面新闻数量

情感值计算

// 每个板块的新闻聚合结果 sentimentSum = Σ signal.strength // 关键词信号 + AI信号 count = 信号总数 positiveCount = count(direction == Positive) negativeCount = count(direction == Negative) newsSentiment = sentimentSum / count // 平均情感值 newsRatio = (positiveCount - negativeCount) / count // 正负比

在逐板块评分之前，先基于大盘环境判断当前市场状态，动态调整各因子权重。

5阶段四：多因子评分公式（FormulaBreakdown）

这是系统的核心评分公式。每个板块独立计算，所有因子相加得到原始预测分。

各因子计算公式

因子公式量级范围
momentumFactor5日动量 × 0.015 + 20日动量 × 0.008约 ±0.15
todayFactor今日涨跌% × 0.015约 ±0.08
sentimentFactornewsSentiment × 0.6 + newsRatio × 0.15约 ±0.75
newsIntensityFactormin(新闻数/8, 1) × 0.08 × sign(sentiment)约 ±0.08
fundFlowFactortanh(flow5dAvg/30) × 0.10约 ±0.10
hotnessFactormin(hotScore/120, 0.08) × trendDirection0 ~ 0.08
techFactortechScore × 0.12（第二遍加入）约 ±0.12
valuationFactor(50 - PE百分位)/50 × 0.08约 ±0.08
crowdingFactor(50 - 拥挤度)/50 × 0.06约 ±0.06
meanReversionPenalty5日动量>8%: -(mom5d-8)×0.025
20日动量>20%: -(mom20d-20)×0.008		负值惩罚

综合评分公式

rawForecast = momentumFactor + todayFactor + sentimentFactor + newsIntensityFactor + fundFlowFactor + hotnessFactor + meanReversionPenalty + techFactor + valuationFactor + crowdingFactor // 数据质量权重 [0.55, 1.00] qualityWeight = 0.55 + 0.45 × (qualityScore / 100) // 来源一致性权重 [0.70, 1.00] consistencyWeight = 0.70 + 0.30 × (consistencyScore / 100) forecastFinal = clamp(rawForecast × qualityWeight × consistencyWeight, -1.0, +1.0)

置信度计算

confidenceBase = clamp( 0.15 × |forecast| // 评分越极端越自信 + 0.40 × min(新闻数/5, 1) // 新闻覆盖越多越自信 + 0.25 × min(K线数/20, 1) // 趋势数据越多越自信 + 0.20 × min(hotScore/40, 1) // 热度越高越自信 , 0.10, 0.95) confidence = clamp(confidenceBase × 质量修正 × 一致性修正, 0.10, 0.95)

数据质量评分

qualityScore = ( K线覆盖率 × 0.30 // min(K线数/60, 1) + 资金流覆盖率 × 0.20 // min(流量点数/20, 1) + 估值覆盖率 × 0.20 // PE/PB/市值至少一项有值 + 新闻覆盖率 × 0.20 // min(新闻数/6, 1) + 多周期覆盖率 × 0.10 // (周线+月线) / 2 ) × 100 // 范围 [0, 100]

决策阈值

评分范围Action趋势标签
≥ +0.30 且 5日动量>1.5%增配强势看多
≥ +0.15偏多
-0.12 ~ +0.15持有方向不明 / 横盘震荡
≤ -0.12减仓偏空
≤ -0.25 且 5日动量<-1.5%强势看空

7阶段六：趋势生命周期系统（核心升级）

这是系统从"趋势跟随"升级为"趋势理解"的核心。在多因子评分之后运行，可以覆盖纯动量追随的建议。

7.1 趋势阶段识别（TrendStageDetector）

对 7 个阶段分别计算匹配分数，取最高者为当前阶段：

阶段关键指标最高分
底部吸筹价格在年低位25%以下 (+30) / MA60走平 (+20) / 波动率收缩<0.8 (+20) / 温和放量 (+15)85
初期突破价格上穿MA60 (+35) / MA多头+MA20斜率>0.3% (+25) / 放量>1.5倍+涨2% (+20)90
主升阶段MA多头+MA20上行 (+25) / 偏离MA60 3%-20% (+25) / 回撤健康>60 (+20) / 量价配合 (+15)95
加速冲顶偏离MA60>15% (+25) / 波动率放大>1.5x (+25) / MA5远离MA20超3% (+20) / 年高位80%+ (+15)85
高位派发派发风险>30% (+30) / 缩量上涨/放量下跌 (+25) / 年高位+MA20走平 (+20) / 波动率>2x (+15)90
下跌趋势MA空头排列 (+30) / MA20斜率<-0.3% (+25) / 偏离MA60<-5% (+20) / 低位 (+15)90
横盘整理MA20走平|<0.15% (+30) / 波动率适中 (+20) / 中间位置 (+15)65
stageConfidence = 最高阶段分数 / 所有阶段总分 × 100%

7.2 趋势健康度（TrendHealthAnalyzer）

维度计算方式权重
结构评分MA排列（多头80/部分55/空头15/其他40）+ MA20斜率修正（>1%→+15, <-0.5%→-15）25%
动量质量近20日上涨天数占比×50 + 涨跌幅正面比例×5020%
回撤健康近30日回撤次数中，回踩MA20后企稳(跌幅<3%)的比例×10020%
量价配合基准50 + 涨配放量×6 - 涨配缩量×8 - 跌配放量×1020%
波动率合理性近10日/近30日标准差比值：<0.7→50, <1.3→85(健康), <2.0→55, >2.0→2515%
sustainability = 结构×0.25 + 动量×0.20 + 回撤×0.20 + 量价×0.20 + 波动×0.15 // 范围 [0, 100]，>65 = 趋势健康, <40 = 趋势恶化

7.3 过热检测（OverheatDetector）

维度计算方式权重
情绪过热新闻数>15: +(n-10)×3 / sentiment>0.3: +sent×60 / 热度>80: +(hot-70)×1.520%
成交量高潮(近5日最大量/30日均量 - 1) × 3020%
波动率爆发(近5日最大|涨跌|/20日均|涨跌| - 1) × 2515%
MA偏离max(偏离MA60% - 5, 0) × 420%
换手率风险近5日日均振幅% × 1010%
拥挤度风险直接使用 crowdingIndex15%
compositeOverheat = Σ(各维度 × 权重) // 范围 [0, 100]，>65 = 过热风险, >40 = 需关注

7.4 可解释推理链（ExplainabilityEngine）

// 趋势强度：基于阶段类型 + 置信度 if 主升/加速 → trendStrength = 70 + confidence × 0.3 if 初期突破 → trendStrength = 55 + confidence × 0.2 if 下跌趋势 → trendStrength = 15 if 高位派发 → trendStrength = 35 else → trendStrength = 40 // 趋势质量 = 健康度的可持续性评分 trendQuality = sustainability // 耗竭风险：三维综合 exhaustionRisk = compositeOverheat × 0.5 + distributionRisk × 0.3 + (100 - breadthHealth) × 0.2

7.5 投资状态判定（InvestmentState）

状态判定条件（优先级从高到低）含义
下跌回避阶段 = 下跌趋势不宜参与
派发警告阶段 = 高位派发 且 过热>50主力出货
过热风险过热>65情绪泡沫
末期动量阶段 = 加速冲顶 且 耗竭>50惯性但风险累积
早期趋势机会阶段 = 初期突破/底部吸筹 且 可持续性>50风险收益比好
健康上涨趋势阶段 = 主升阶段 且 可持续性>55趋势中继
防御持有其他所有情况观望等待

策略买入信号卖出信号最大持仓
MACD金叉策略DIF上穿DEADIF下穿DEA20天
RSI超卖反弹RSI6从<25上穿25RSI6>7520天

// 回测统计 winRate = wins / totalTrades × 100% avgReturn = Σ(每笔收益%) / totalTrades maxDrawdown = max(peak − trough) / peak × 100% // 当前信号：基于最新K线重新评估（非历史最后一笔） if 仍持仓 → "持仓中" else if 最新K线触发买入 → "买入信号" else if 最新K线触发卖出 → "卖出信号" else if 前一根K线买入 → "近期买入" else → "观望"

ATR(14) 动态止盈止损

// 14日平均真实波幅 TR[i] = max(High-Low, |High-PrevClose|, |Low-PrevClose|) ATR14 = mean(TR[last14]) atr14Pct = ATR14 / lastClose × 100% if atr14Pct > 0.5%: // 有足够波动率 stopLoss = clamp(min(支撑位距离-1%, -ATR×2.0), -12%, -1.5%) takeProfit = clamp(max(阻力位距离+1%, +ATR×2.5), +2%, +20%) else: // 低波动回退 stopLoss = clamp(支撑位距离-2%, -8%, -3%) takeProfit = clamp(阻力位距离+2%, +3%, +15%)

9阶段八：最终决策与输出

在多因子评分和趋势生命周期分析完成后，系统进行最终的决策校正：

InvestmentState 覆盖 Action

// 趋势理解覆盖纯动量追随 if 状态 == 派发警告 || 下跌回避: 增配 → 减仓 // 即使动量强也要降级 if 状态 == 过热风险 || 末期动量: 增配 → 持有 // 不追高 if 状态 == 早期趋势机会: 持有 → 增配 // 条件：forecastFinal > 0.05

板块轮动信号注入

// 轮动评分 = 动量加速 × 0.5 + 资金加速 × 0.3 + 5日动量 × 0.2 rotationScore = momentumDelta × 0.5 + fundFlowAccel × 0.3 + mom5d × 0.2 // 轮入条件 isRotatingIn = momentumDelta > 1.0 且 mom5d > 0.5 // 轮出条件 isRotatingOut = momentumDelta < -1.0 且 mom5d < -0.5

累计收益计算

// 从用户配置的"首次收益日"开始追踪 startPrice = 找到 firstEarnDate 当日或之后的第一根K线收盘价 if firstEarnDate > 今天 → return = 0% // 还没开始 if K线中找不到 → startPrice = 最后一根K线 // 保守回退 cumulativeReturn = (currentPrice - startPrice) / startPrice × 100% trackingDays = startDate 到今天的天数

最终输出字段（SectorSnapshot 关键字段）

评分决策
forecastScore [-1,+1] → action (增配/持有/减仓) → trendSummary (强势看多~强势看空)
趋势理解
trendStageResult (7阶段) → trendHealth (5维健康度) → explanation (推理链+InvestmentState)
风险控制
overheat (6维过热) → stopLossPct / takeProfitPct (ATR动态) → action覆盖
策略验证
backtestResults (5策略胜率) → bestStrategyCurrentSignal → investmentRecommendation

10UI 展示层：板块详情呈现

用户点击板块后看到的详情页，按以下顺序呈现计算结果：

  1. 投资结论区
    操作标签（买入/持有/减仓/止损） ← strategy.actionLabel
    综合评分 ← forecastScore / 置信度 ← confidence / 趋势 ← trendSummary
    三张卡片：预期收益(止盈%) / 最大风险(止损%) / 累计收益(%) ← ATR计算 + 累计追踪
    短期/中期/长期展望文本 ← strategy.shortTermView/mediumTermView/longTermView
    操作建议 ← strategy.operationAdvice
    看多/看空因素列表 ← positiveFactors / negativeFactors
    AI深度分析 ← aiAnalysis（若启用）
    我的持仓建议 ← personalAdvice（含趋势阶段信息）
  2. 趋势生命周期分析区
    趋势阶段标签（7态） ← trendStageResult.stageName
    投资状态标签（7态） ← explanation.stateName
    阶段置信度% ← trendStageResult.stageConfidence
    健康度 5 维卡片（结构/动量/回撤/量价/波动/可持续） ← trendHealth
    三维评分条（趋势强度/趋势质量/耗竭风险） ← explanation
    推理链文本（强度/可持续性/风险原因） ← explanation.*Reason
    结论文本 ← explanation.conclusion
    资金结构卡片（模式/连续流入天数/机构特征/投机风险） ← flowStructure
    风险警告区 ← overheat.warnings
    过热指数进度条 ← overheat.compositeOverheat
  3. 数据图表区
    趋势图 × 3（近3月/6月/1年）← QPainter 绘制
    K线图 × 3（日K/周K/月K）← OHLC 蜡烛图
    核心指标卡片（动量/估值分位/拥挤度） ← tech / pePercentile / crowdingIndex
    成交量面板 / MACD面板 / 资金流面板 ← QPainter 绘制
    策略回测表（5策略：交易次数/胜率/平均收益/最大回撤/当前信号） ← backtestResults
  4. 关键事件与新闻区
    未来事件前瞻 ← upcomingEvents
    事件驱动涨跌 ← eventDrivenMoves
    新闻列表（按时间倒序，含情感标签+可点击链接） ← newsEntries
