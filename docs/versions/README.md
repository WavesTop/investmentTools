# InvestInsight 版本总览

最后更新：2026-06-26

本文档用于快速理解 InvestInsight 每个版本的主要变化。需要查看具体设计稿、开发计划、截图或发布说明时，再进入对应版本目录。

## 版本速览

| 版本 | 核心目标 | 用户可感知更新 | 关键文档 | 状态 |
| --- | --- | --- | --- | --- |
| `v1.0` | 形成可安装、可启动的软件形态 | Windows / macOS 打包脚本、应用图标、启动说明和基础发布结构 | `v1.0/README.md`、`v1.0/release/PACKAGING.md` | 已落地 |
| `v2.0` | 建立工作台 UI 与事件影响引擎方向 | 左侧工作台、界面设计稿、板块详情设计、事件传导能力规划 | `v2.0/README.md` | 已落地，部分能力继续演进 |
| `v2.1` | 补完事件传导引擎 | 事件状态、时间节点、路径规则、板块映射、评分与追踪机制 | `v2.1/README.md` | 已落地 |
| `v2.2` | 提升内容可读性与 AI 协同分析 | 关键事件解释、事件证据链、理由去重、板块机会表格、详情趋势图与截图验证 | `v2.2/README.md` | 已落地 |
| `v2.3` | 建立推荐生命周期与信任机制 | 避免追高式推荐，增加回调观察、过热不追、风险预警和信号复盘 | `v2.3/README.md` | 已落地 |
| `v2.4` | 多周期技术点位与推荐模型 | 从今日涨幅推荐升级为支撑/阻力、买入观察区、止盈减仓区、板块详情聚焦布局和几个月维度的推荐判断 | `v2.4/README.md` | 第一轮已落地 |

## 演进主线

1. `v1.0` 解决“软件能发布和运行”的问题。
2. `v2.0` 到 `v2.1` 解决“新闻事件如何影响板块”的问题。
3. `v2.2` 解决“用户能否看懂系统为什么这么判断”的问题。
4. `v2.3` 解决“推荐是否有连续性、是否避免涨了才推荐”的问题。
5. `v2.4` 计划解决“推荐是否具备合理买卖点、持有周期和风险收益比”的问题。
