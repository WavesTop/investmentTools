# v2.5 工作台新界面设计说明

最后更新：2026-06-26

## 设计选择

使用 `DESIGN.md UI` 工作流后，v2.5 选择自定义 **Insight Workbench** 风格：

- 借鉴 `linear.app` 的冷静、精确、低噪声导航和页面层级。
- 借鉴 `sentry` 的事件队列、风险分级和异常追踪信息组织。
- 借鉴 `clickhouse` 的数据密度和高频扫读表格。
- 借鉴 `coinbase` 的金融信任感，但不使用其品牌色或品牌视觉。

当前项目是投研工作台，不适合 Apple/Nike/Stripe 这类营销型页面，也不适合 Binance/Kraken 那种强交易刺激风格。新界面应更像“事件驱动的研究终端”，而不是“涨跌排行列表”。

## 当前界面问题归纳

1. 左侧导航已稳定，但主内容区仍像长文档，缺少页面级布局节奏。
2. 总览页信息从上到下堆叠，用户需要滚动很久才能看到板块机会。
3. 事件雷达和总览的事件表达相似，缺少状态、路径、观察点的层级区分。
4. 板块机会表格数据很全，但列密度和主次关系不够清晰。
5. 板块详情保留了大量信息，本质上是长页面；设计稿必须为趋势图、购买建议、事件分析、预测矩阵、证据层、新闻证据和数据质量预留完整纵向空间，不能只按首屏高度实现。
6. AI 助手首屏右侧空白过多，配置页控件大小和排列仍有表单拼接感。

## 新界面原则

- 每页首屏只回答一个核心问题。
- 每个长列表只展示摘要，把完整证据放到详情或下方证据层。
- 事件必须从“新闻标题”升级为“事件状态 + 影响路径 + 下一观察点”。
- 板块表格要优先服务筛选，不承担长解释。
- 详情页要让用户先看买卖点，再看原因和证据。

## 实施建议

1. Phase 1：先改 `AppTheme` token 和左侧/顶部基础样式。
2. Phase 2：重排总览页，减少长新闻堆叠，加入事件卡和 Top 板块摘要。
3. Phase 3：重排事件雷达，把事件队列、传导路径、风险条件分清。
4. Phase 4：重排板块机会表格，稳定列宽和行高。
5. Phase 5：细化板块详情长页，截图验收至少覆盖首屏、事件/预测区、证据层和新闻证据底部。
6. Phase 6：重排 AI 助手与配置页，并统一表单控件尺寸。

每个阶段都需要 UI smoke 和截图确认。
