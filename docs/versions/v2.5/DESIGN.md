# InvestInsight v2.5 DESIGN.md

最后更新：2026-06-26

## 设计来源

本设计系统参考 `VoltAgent/awesome-design-md` 的 DESIGN.md 工作流，但不直接复制单一品牌。最终选择自定义方向：

- `linear.app`：克制、精确、低噪声的工作台层级。
- `sentry`：事件队列、风险分级、异常追踪的可扫描结构。
- `clickhouse`：高密度表格、技术指标和分析面板的信息承载方式。
- `coinbase`：金融产品需要的可信、清晰、低误导表达。

最终风格命名为 **Insight Workbench**：冷静、可信、数据密集，但不压迫用户；优先帮助用户判断“现在该看什么、为什么、下一步验证什么”。

## 设计原则

1. 决策先于解释：每个页面首屏先给当前状态、首要风险和下一动作，再展开证据。
2. 证据可追溯：事件、新闻、资金、技术指标和 AI 观点必须能被用户追到来源或原因。
3. 风险显性：不要只突出上涨推荐；回调观察、过热不追、风险预警要有同等视觉权重。
4. 高密度但不拥挤：表格行距收敛，列宽稳定，重点字段用色块和数字层级区分。
5. 避免营销感：不使用大 hero、渐变背景、装饰插画、过度圆角或漂浮卡片。
6. 适配 Qt：优先使用 Qt Widgets 和 QTextBrowser 可靠支持的布局，复杂 CSS 只作为未来 WebEngine 方向。

## 信息架构

左侧导航保持常驻：

- 总览
- 事件雷达
- 板块机会
- 策略跟踪
- AI 助手
- 配置

顶部状态条只展示当前页面标题、运行状态、数据时间、板块数量和规则/AI 状态，不放置重复导航。

主内容区采用三层结构：

1. **状态带**：刷新控制、市场状态、当日风险、AI/规则状态。
2. **工作区**：页面核心内容，例如事件队列、板块表格、趋势图。
3. **证据区**：新闻、指标、资金流、回测、数据质量和可追溯说明。

颜色规则：

- A 股语境保持红涨蓝跌。
- 风险用橙色，不用红色抢占涨跌语义。
- AI 只作为解释辅助色，不压过规则评分。
- 大面积背景只用白、浅灰、深导航三类，不做彩色渐变。

| Section title | 17 | 700 | 分区标题 |

间距：

- 页面外边距：24px。
- 面板内边距：16px。
- 表格行高：38-44px。
- 卡片圆角：6px，不超过 8px。
- 页面区块间距：16px。

## 组件

### 左侧导航

- 宽度 236px。
- 深色背景，品牌区固定在顶部，数据状态固定在底部。
- 选中项使用浅色块，文字深蓝。
- 不使用图标也可接受；若加入图标，必须小于 16px 且不增加噪声。

### 顶部状态条

- 高度 64px。
- 左侧：页面标题 + 一句话定位。
- 右侧：数据时间、板块数量、AI/规则状态、刷新状态。
- 避免放置重复的 AI/配置按钮。

### 状态带

用于总览和关键页面顶部：

- 左侧展示市场状态和风险等级。
- 中间展示当日关键事件和下一观察点。
- 右侧放“开始分析”和 AI 开关。
- 状态带不做巨型卡片，保持 72-96px 高。

### 指标卡

- 用于市场温度、板块数量、事件数量、资金流。
- 每张卡只放一个主数字和一个解释短语。
- 不使用四周厚边框；用浅底和细边框。

### 事件卡

事件展示采用三段：

1. 标题：改写为人能理解的判断句。
2. 元信息：来源、时间、状态、可信度。
3. 影响路径：事件 -> 变量 -> 板块 -> 观察点。

必须包含“查看来源”或可点击标题的视觉暗示。

### 板块表格

表格必须优先扫读：

- 固定列：板块、今日、5日、20日、事件、技术、资金、点位、建议、下一观察。
- 删除长段风险文案，改为风险标签和 hover/详情。
- 表头吸顶作为未来增强；Qt QTextBrowser 当前可先保持静态表头。
- 行高 40px，长文本只显示一行摘要。

### 板块详情

首屏布局：

- 左侧 62%：趋势图与技术图表。
- 右侧 38%：决策摘要、观察买入区、止损失效位、止盈减仓区、风险收益比。
- 图表下方：事件影响、预测矩阵、证据层。

详情页禁止把新闻长列表放到首屏。

### AI 助手

- 左侧上下文面板压缩为“当前市场、重点事件、强弱板块、风险板块”。
- 右侧对话区更像投研问答，不做空白欢迎页。
- 快捷问题使用小型按钮组，不铺满大面积卡片。

### 配置

- 采用两列：AI 接入 + 我的持仓。
- 后台刷新和数据源健康放到下方。
- 表单行高和输入框宽度保持一致，避免控件大小跳动。

## 页面设计

### 总览

目标：用户 10 秒内知道今天市场是否适合行动。

结构：

1. 状态带：市场风险、刷新控制、下一观察点。
2. 市场仪表：市场温度、北向资金、指数涨跌、事件数量。
3. 关键事件雷达：3-5 条最重要事件，每条带来源和影响路径。
4. 板块机会与风险：只展示 Top 8，突出建议、风险和下一观察。
5. AI 与规则摘要：说明 AI 是否介入、规则是否存在分歧。

### 事件雷达

目标：把新闻变成可追踪事件，而不是新闻列表。

结构：

1. 事件状态统计：预期、已发生、失效、待验证。
2. 结构化事件时间线。
3. 关键事件队列：事件标题、证据、状态、观察点、影响说明。
4. 事件传导路径：事件 -> 变量 -> 板块 -> 衰减/失效条件。
5. 风险与失效条件。

### 板块机会

目标：让用户像看交易终端一样快速筛选。

结构：

1. 筛选条：视图模式、搜索、趋势、建议、排序。
2. 总结条：本轮扫描数量、当前排序、数据质量。
3. 高密度表格。
4. 选中行详情预览作为未来增强。

### 策略跟踪

目标：解决“昨天推荐、今天下跌就消失”的信任问题。

结构：

1. 跟踪状态：推荐中、回调观察、风险预警、失效移除。
2. 推荐生命周期列表。
3. 信号复盘：触发原因、后续表现、失效条件。
4. 市场操作建议。

### 板块详情

目标：一个板块从“是否行动”到“为什么”的完整决策页。

结构：

1. 决策摘要。
2. 趋势图与点位计划。
3. 相关事件分析。
4. 预测矩阵。
5. 证据层：核心评分、AI 解读、资金流、技术指标、回测、新闻证据、数据质量。

### AI 助手与配置

目标：AI 助手更像上下文问答，配置更像工作台设置。

AI 助手：

- 左侧上下文压缩但保持可问。
- 右侧对话区首屏给出“当前可以问什么”，不要大面积空白。

配置：

- AI Key、持仓、刷新提醒、数据源健康同屏。
- 保存按钮固定在页面底部。

## Qt 落地约束

- `AppTheme` 中落地颜色 token。
- `QTextBrowser` 渲染页优先使用 table、基础 inline style 和现有 class，避免复杂 grid/flex。
- 大表格需要稳定列宽、固定行高和单行摘要。
- 复杂交互例如吸顶表头、折叠证据层、图表联动，未来可评估 Qt WebEngine，但需同步打包说明。
- UI 改动后必须运行 `tools/verify_ui_smoke.ps1` 并截图确认。

## 不做什么

- 不做营销落地页。
- 不做大面积渐变、玻璃拟态或装饰图形。
- 不把 AI 文案放在规则评分前面。
- 不隐藏风险，只突出上涨机会。
- 不为了视觉简洁删除数据质量、新闻证据、资金流或回测信息。
