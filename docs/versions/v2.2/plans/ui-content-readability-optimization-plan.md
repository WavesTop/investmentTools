# UI 内容可读性优化开发计划

最后更新：2026-06-23

## 背景

v2.1 已经把新闻结构化为事件，并能解释事件状态、影响路径和板块映射；但当前界面把这些结果展示得过于“原始”。从总览和板块机会截图看，用户仍然需要二次理解：事件标题不像事件摘要，理由和风险大量重复，表格文字过密，部分关键内容缺少原始链接和趋势图支撑。

v2.2 的目标不是重新发明分析引擎，而是把现有分析结果改造成更可信、更可读、更容易行动的产品表达。同时引入 AI 协同分析层，让 AI 负责理解、归纳、去重、解释和交叉验证，但不直接接管核心评分或买卖动作。

## 当前问题诊断

### 关键事件雷达

- 事件数量偏少，容易让总览显得信息不足。
- 标题直接使用新闻标题或规则化标题，缺少“这件事是什么、为什么重要、影响谁”的解释。
- 标题不可点击，用户不能跳转到原始网页核对来源。
- 缺少来源、发布时间、事件状态、受影响板块和下一观察点的稳定展示。

### 板块机会与风险

- 理由和风险文案高度重复，例如“强势看多”“近 5 日涨幅较大，存在回调风险”等句式频繁出现。
- 文案排列过密，用户很难一眼区分今日涨幅、事件催化、趋势状态、操作动作和风险。
- 列表页承载了过多解释性内容，应该先服务快速扫读，把详细解释留给板块详情页。

### 事件雷达

- 与总览存在同类表达问题：事件标题不够直白，事件原因和传导路径需要用户自己翻译。
- 多条事件容易出现重复表达，削弱用户对系统真实性和分析质量的信任。
- 当前更像“事件数据列表”，还没有形成“事件状态追踪台”。

### 策略跟踪与事件雷达

- 两者存在重叠：都可能展示事件、影响板块、观察点和风险。
- 建议 v2.2 先明确职责，而不是立即强行合并：
  - 事件雷达：负责“事前和事中”的事件识别、状态、证据、影响路径。
  - 策略跟踪：负责“信号形成之后”的动作、持仓适配、后续表现和失效条件。
- 如果实现后仍出现大量重复，再把左侧导航合并为“事件与策略”，内部保留两个视图。

### 板块机会页面

- 表格里的“风险提示”适合详情页，不适合列表页。
- 列表页应增加用户熟悉的量化指标，例如 MACD、RSI、KDJ、均线状态、成交量、资金流、数据质量和建议动作。
- 用户点击板块后，再进入详情页查看理由、风险、事件证据和趋势图。

### 板块详情页面

- 事件标题、理由和风险同样需要去重、直白化和证据化。
- 之前用于判断趋势的趋势图需要恢复或确认可见，详情页必须保留图表支撑。
- 详情页应成为“解释为什么”的页面，包含事件证据、影响路径、技术图表、资金流和失效条件。

### AI 介入深度

- 当前 AI 主要用于新闻归因和 Top N 板块深度分析，核心评分、事件路径、板块映射和风险控制仍以规则与量化为主。
- 现有 AI 输出更多是“补充理由”，还没有系统参与事件标题改写、理由去重、影响路径自然语言解释和规则分歧提示。
- v2.2 应扩大 AI 使用范围，但定位为协同分析层，而不是直接替代规则引擎。
- AI 需要输出结构化字段，避免只生成一段不可测试的长文本。

## 设计原则

- 先可读，再完整：列表页只展示最重要的判断，详情页承载完整解释。
- 证据优先：可点击标题、来源、发布时间、证据链接和可信度要稳定出现。
- 减少套话：同一页面内相同理由和风险需要合并、降权或换成更具体的因子解释。
- 页面职责清晰：总览用于快速判断，事件雷达用于追踪事件，板块机会用于横向比较，策略跟踪用于信号后的持有和复盘，板块详情用于深度解释。
- AI 辅助解释，不直接裁决：AI 可以改写标题、生成摘要、解释路径和指出分歧，但不能直接覆盖 `forecastScore`、`eventCatalystScore` 或最终动作。
- 每次 UI 或内容显示改动都需要截图确认，避免只通过 smoke 测试却漏掉真实布局问题。

## 页面调整方案

### 总览

- 关键事件雷达改为事件卡片或紧凑列表，每条事件包含：
  - 用户可读标题。
  - 可点击原文链接。
  - 来源、发布时间、事件状态和可信度。
  - 影响板块标签。
  - 一句话解释“为什么重要”。
  - 下一观察点。
- 默认展示 5 到 8 条高质量事件，数量不足时显示数据不足原因，而不是静默变少。
- 板块机会与风险改成扫读型表格或卡片：
  - 板块、今日涨幅、事件分、趋势分、建议动作。
  - 首要理由一条。
  - 首要风险一条。
  - 数据质量。
- 长文本限制行数，详情通过点击板块进入。

### 事件雷达

- 按事件状态分组：传闻、预期、排期、已发生、修正、失效。
- 每个事件展示证据链：原文标题、来源、时间、可信度、关联板块、影响方向。
- 对相同事件或相同来源标题做去重聚合，展示“多来源确认”而不是重复多行。
- 事件标题要从新闻标题加工成事件表达，例如“美联储 6 月会议前降息预期升温”，而不是堆叠关键词。

### 板块机会

- 删除列表表格中的“风险提示”列。
- 增加常用技术和资金字段：
  - MACD 状态。
  - RSI 区间。
  - KDJ 风险。
  - 均线趋势。
  - 成交量变化。
  - 主力资金或北向资金。
  - 事件催化分。
  - 数据质量。
- 表格字段优先支持排序和筛选，风险解释进入板块详情。

### 策略跟踪

- 保留“信号之后”的职责：
  - 当前建议动作。
  - 信号触发原因。
  - 持有观察点。
  - 止盈止损或失效条件。
  - 事件后的 1/3/5/10 日表现。
- 不再重复展示完整事件列表，只引用事件雷达中的关键事件。

### 板块详情

- 恢复或确认趋势图显示，至少包含趋势图、K 线、成交量和 MACD。
- 顶部给出清晰结论：建议动作、适用周期、置信度、主要驱动。
- 事件内容使用“事件 -> 路径 -> 板块影响 -> 验证点”的结构。
- 风险内容分为技术风险、事件风险、资金风险和数据风险，避免重复模板句。
- 新闻证据标题可点击打开原文。

### AI 协同分析层

- 在规则引擎完成事件抽取、路径映射和板块初筛后调用 AI，输入结构化事件、证据标题、来源、URL、板块指标和规则结论。
- AI 输出结构化 JSON，不直接输出不可解析的大段文字。
- 建议输出字段：
  - `readableTitle`：用户可读事件标题。
  - `summary`：一句话解释事件是什么。
  - `whyItMatters`：为什么会影响市场或板块。
  - `impactPath`：事件到板块的传导路径。
  - `affectedSectors`：受影响板块和方向。
  - `primaryReason`：板块首要机会理由。
  - `primaryRisk`：板块首要风险。
  - `nextCheckpoint`：下一观察点。
  - `confidence`：AI 对该解释的置信度。
  - `sourceRefs`：引用的新闻来源或证据编号。
  - `disagreementNotes`：AI 与规则判断不一致时的分歧说明。
- AI 失败、关闭、超时或 JSON 解析失败时，系统继续使用规则引擎结果。
- 页面需要明确标识 AI 参与的位置，让用户知道哪些内容来自规则、行情、资金、技术面，哪些是 AI 总结。

## 数据与实现切片

### 切片 1：内容模型补齐

目标：让 UI 渲染层拿到可读标题、原文链接、来源、时间、事件状态、影响板块和首要理由。

候选改动：
- 检查 `MacroEvent`、`MacroEventEvidence`、`SectorEventImpact` 和 `SectorSnapshot` 是否已经有足够字段。
- 增加 UI 专用的展示摘要生成函数，避免 renderer 自己拼复杂逻辑。
- 增加事件去重和理由去重规则。

验证：
```powershell
cmake --build build --config Release -- /m
.\build\Release\InvestInsightEventSmoke.exe
```

### 切片 2：AI 协同分析层

目标：让 AI 参与事件标题改写、摘要生成、理由/风险去重和规则分歧提示，同时保持核心评分由规则与量化系统主导。

候选改动：
- 扩展 `AIAnalyzer` 的结构化输出能力，新增面向 UI 表达的 prompt 和 JSON 解析。
- 输入只传递必要的结构化事件、板块指标、新闻证据和规则结论，避免让 AI 自由猜测不存在的数据。
- 输出写入独立展示字段或 UI view model，不能直接覆盖 `forecastScore`、`eventCatalystScore` 和最终建议动作。
- 增加 AI 与规则结论冲突的标记，例如“规则偏谨慎，AI 认为事件仍有后续催化”。
- AI 关闭或调用失败时保留现有规则结果和页面结构。

验证：
```powershell
cmake --build build --config Release -- /m
.\build\Release\InvestInsightEventSmoke.exe
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\verify_ui_smoke.ps1
```

测试要点：
- 使用固定 AI JSON 样本验证解析器。
- AI 输出缺字段、字段类型错误或包含无效 JSON 时不能影响主流程。
- AI 分歧提示只能展示，不能自动改写核心评分。

### 切片 3：总览可读性改造

目标：关键事件和板块机会在总览中一眼可读，并支持原文跳转。

候选改动：
- 修改 `DashboardRenderer` 的关键事件雷达。
- 修改总览中的板块机会与风险模块。
- 接入 AI 生成的可读事件标题、`whyItMatters`、首要理由和首要风险。
- 为关键事件和板块行增加 smoke 断言。

验证：
```powershell

```

截图确认：
- 总览首屏，包含 AI 提炼后的事件标题和规则/AI 来源标识。
- 总览滚动到板块机会与风险区域。

### 切片 4：事件雷达去重与证据化

目标：事件雷达从“列表”升级为“事件状态追踪台”。

候选改动：
- 修改 `EventRadarRenderer` 的分组和卡片结构。
- 对重复事件和重复文案进行聚合展示。
- 原文标题支持点击。
- 展示 AI 生成的事件摘要、影响路径解释和分歧说明。

### 切片 5：板块机会表格字段调整

目标：表格成为横向比较工具，不再承担长文本风险解释。

候选改动：
- 修改 `SectorTableRenderer` 字段。
- 删除“风险提示”列。
- 增加 MACD、RSI、KDJ、均线、成交量、资金流、事件分和数据质量字段。
- 首要理由和首要风险来自规则与 AI 去重后的摘要，不展示长段解释。
- 保留点击进入详情页能力。

目标：详情页同时可解释、可核对、可看图。

候选改动：
- 检查 `SectorDetailRenderer` 是否仍嵌入 `ChartRenderer` 输出。
- 若趋势图丢失，恢复趋势图模块。
- 调整事件、理由、风险和新闻证据的排版。
- 增加 AI 协同分析区，展示事件解释、影响路径、规则分歧和下一观察点。

验证：
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\verify_ui_smoke.ps1
```

截图确认：
- 板块详情顶部结论。
- 趋势图和技术指标区域。
- 新闻证据区域。

### 切片 7：策略跟踪与事件雷达职责收束

目标：减少两个页面的重复表达。

候选改动：
- 策略跟踪只展示信号后的动作、持有观察点、失效条件和事后表现。
- 事件列表通过摘要引用，不再重复渲染完整事件雷达内容。
- 如重复仍明显，再考虑导航合并。

## 自动化截图要求

UI 和用户可见内容改动完成后，需要通过自动化入口跳转到目标页面并截图确认。测试不强制模拟鼠标点击，可以新增调试参数、测试接口或 UI smoke 入口直接打开页面、等待分析完成、再截图。

最少截图范围：
- 总览。
- 事件雷达。
- 板块机会。
- 策略跟踪。
- 板块详情。

截图应在交付说明或提交说明中记录路径或验证结论。

## 完成标准

- 关键事件标题可读，并在存在 URL 时可点击打开原文。
- 总览关键事件数量稳定，不足时给出原因。
- 板块机会与风险不再大面积重复模板文案。
- 板块机会表格删除“风险提示”，增加技术和资金字段。
- 事件雷达按状态和证据链组织，重复事件被聚合。
- AI 协同分析层可以输出可读标题、摘要、影响路径、理由/风险去重和规则分歧提示。
- AI 关闭、失败或解析异常时，规则引擎结果仍可完整展示。
- AI 不直接覆盖核心评分和最终动作，只作为解释、归纳和交叉验证层。
- 策略跟踪与事件雷达职责明确，重复内容明显减少。
- 板块详情恢复趋势图，并保留新闻证据、影响路径和风险解释。
- UI smoke、相关核心 smoke 和截图验证通过后再提交本地 commit。
