# 技术点位与多周期推荐模型开发方案

最后更新：2026-06-26

## 背景

当前推荐容易给用户形成“涨了才推荐、跌了就消失”的感受。v2.3 已经加入推荐生命周期，用于保留信号、提示回调观察和风险预警；v2.4 需要进一步补足“什么时候适合观察买入、涨到哪里应该止盈或减仓、为什么这个判断能覆盖未来几周到几个月”的解释能力。

本方案不把系统升级成直接下单工具，也不输出确定性的投资建议。所有点位都应表达为观察区间、失效条件和风险收益比，避免使用“必买”“必涨”“保证卖出”等确定性描述。

## v2.4 目标

1. 为每个推荐板块输出“趋势状态 + 观察买入区 + 止损失效位 + 止盈/减仓区 + 持有周期”。
2. 让推荐至少覆盖短期到中期，也就是 5 到 60 个交易日，重点解释未来几周到几个月是否仍有逻辑。
3. 将“今日涨幅”降级为证据之一，而不是推荐决定因素。
4. 对上涨板块提示追高风险，对下跌板块提示是否进入回调观察、趋势破坏或风险释放阶段。
5. 让用户能区分“可观察”“可分批”“等待突破确认”“过热不追”“风险预警”“逻辑失效”。

## 技术点位分析框架

### 数据输入

- 日 K、周 K，后续可补月 K。
- 均线：MA5、MA10、MA20、MA60、MA120、MA250。
- 动量指标：MACD、RSI、KDJ。
- 波动指标：ATR、BOLL。
- 量能指标：成交额、成交量、量比、近 5/20/60 日均量。
- 结构点：近期高点、近期低点、跳空缺口、平台上沿、平台下沿。
- 事件证据：政策、产业、商品价格、资金、海外宏观、财报或供需变化。

### 趋势状态

| 状态 | 判断重点 | 推荐表达 |
| --- | --- | --- |
| 上升趋势 | 价格在中期均线上方，均线多头排列，回调不破关键支撑 | 回调到支撑区观察，突破后等待回踩确认 |
| 下降趋势 | 价格在中期均线下方，反弹受阻，低点继续下移 | 不追反弹，等待止跌结构和量能确认 |
| 震荡区间 | 价格在箱体内反复，趋势强度不足 | 箱体下沿观察，上沿减仓，不做趋势外推 |
| 突破确认 | 放量突破平台或前高，并能站稳 | 关注回踩不破后的分批机会 |
| 过热拉升 | 价格远离均线，RSI 或乖离率偏高 | 保留逻辑但标记过热，不建议追高 |
| 跌破失效 | 跌破关键支撑且无法收回 | 降级为风险预警或逻辑失效 |

### 买入观察区

买入观察区不应是一个单点，而是一个价格区间。系统可以按以下优先级生成：

1. 近期有效支撑位：近 20/60/120 个交易日局部低点、平台下沿、前期高点突破后的回踩位。
2. 动态支撑：MA20、MA60、BOLL 中轨或下轨，结合 ATR 给出缓冲区。
3. 量能密集区：如果后续能拿到筹码或成交分布，用成交密集区作为补充。
4. 事件确认点：重大事件尚未落地时，买入区需要附带“等待事件确认”的条件。
5. 风险收益比：观察买入区到止盈区的潜在收益，应明显大于到止损位的潜在亏损。

### 止盈与减仓区

止盈区同样应表达为区间，而不是固定点位：

1. 前高、平台上沿、通道上轨和历史密集成交区。
2. 放量突破后的量度目标，例如箱体高度或形态高度，但必须标注“形态确认后才启用”。
3. BOLL 上轨、ATR 移动止盈和 RSI 过热区，用于提示分批减仓。
4. 当事件预期兑现但价格已经提前大涨时，提示“利好兑现风险”。

### 失效与风控

每条推荐必须同时给出失效条件：

- 跌破关键支撑并连续 2 到 3 个交易日无法收回。
- 事件预期被证伪，例如政策不及预期、商品价格反向、订单或财报不及预期。
- 市场环境恶化，例如指数破位、北向或主力资金持续流出、板块轮动退潮。
- 风险收益比低于阈值，例如小于 1.5，不应给出积极推荐。

## 多周期推荐模型

### 周期定义

| 周期 | 观察窗口 | 主要依据 | 输出重点 |
| --- | --- | --- | --- |
| 短期 | 5 到 20 个交易日 | 事件催化、技术突破、量价配合、波动风险 | 能否介入、是否追高、短线止盈区 |
| 中期 | 20 到 60 个交易日 | 趋势延续、资金持续性、行业景气和政策兑现 | 是否值得持有、回调是否加分 |
| 长周期观察 | 60 到 120 个交易日 | 周线结构、产业周期、业绩验证和估值位置 | 是否只是短炒，还是有阶段性主线 |

### 评分拆分

推荐分不再由单一涨幅或单一 AI 结论决定，拆为以下维度：

| 维度 | 含义 |
| --- | --- |
| `TrendScore` | 趋势方向和强度，来自均线、MACD、结构高低点。 |
| `TimingScore` | 当前是否适合介入，重点看是否接近支撑、是否远离均线、是否过热。 |
| `RiskRewardScore` | 买入观察区、止损位和止盈区之间的风险收益比。 |
| `EventScore` | 新闻事件强度、状态、时间节点和影响路径。 |
| `PersistenceScore` | 事件和趋势能否覆盖未来数周到数月。 |
| `CrowdingRisk` | 涨幅过大、RSI 过热、成交拥挤或短期资金过度流入。 |
| `EvidenceQuality` | 数据质量、来源数量、AI 解释一致性和是否存在反向证据。 |

### 推荐动作

| 动作 | 触发条件 | 用户看到的表达 |
| --- | --- | --- |
| 观察 | 逻辑存在，但技术位或事件状态未确认 | 有逻辑，等待价格回到观察区或事件确认 |
| 回调分批 | 趋势仍在，价格接近支撑且风险收益比合格 | 接近观察区，可分批关注，跌破支撑则失效 |
| 突破确认 | 放量突破关键阻力并站稳 | 等待回踩确认，止损放在突破位下方 |
| 持有跟踪 | 已在推荐生命周期内，趋势未破坏 | 保留逻辑，关注止盈区和风险提示 |
| 过热不追 | 逻辑强但价格远离均线或风险收益比不足 | 不是删除推荐，而是提示等待回调 |
| 风险预警 | 趋势或事件出现反向证据 | 降低仓位或停止新增，观察是否收回支撑 |
| 逻辑失效 | 支撑跌破、事件证伪或趋势破坏 | 移出积极推荐，并保留复盘记录 |

## 实施阶段

### Phase 1：指标与点位模型

- 新增技术点位模型，例如 `PriceLevelPlan`。
- 记录支撑区、阻力区、观察买入区、止损位、止盈区、风险收益比和证据来源。
- 编写纯逻辑测试，用构造 K 线覆盖上涨、下跌、震荡、突破和过热场景。

### Phase 2：K 线结构分析器

- 基于日 K / 周 K 计算均线、MACD、RSI、KDJ、BOLL、ATR。
- 识别局部高低点、箱体、突破、回踩和跌破。
- 避免只因今日涨幅高就直接给高分。

### Phase 3：推荐评分接入

- 将 `TrendScore`、`TimingScore`、`RiskRewardScore`、`PersistenceScore` 接入现有推荐流水线。
- 与 v2.3 推荐生命周期联动，保留过热不追、回调观察、风险预警和逻辑失效状态。
- AI 只作为解释与交叉验证层，不直接覆盖规则层分数。

### Phase 4：界面表达

- 板块机会表格增加趋势状态、观察区、止盈区、风险收益比和周期标签。
- 板块详情恢复并强化趋势图，在图上展示支撑/阻力/观察区。
- 总览页不再只展示“今日涨幅 + 增配”，而是展示“为什么现在不一定适合买”。

### Phase 5：验证与复盘

- 用历史 20/60/120 个交易日回看推荐信号，记录最大回撤、命中止盈、触发止损和持有期表现。
- 对同花顺板块口径继续保留实时涨幅校验。
- 用固定样本板块进行截图验证：半导体、有色金属、锂电池，以及一个下跌或震荡板块。

## 验收标准

- 每个积极推荐都必须包含观察买入区、止损失效位、止盈或减仓区。
- 没有合格风险收益比时，不能输出“增配”，只能输出观察或过热不追。
- 今日涨幅不能单独决定推荐动作。
- 下跌板块不能简单消失，必须进入风险预警、回调观察或逻辑失效复盘。
- UI 文案必须明确区分“观察区间”和“交易指令”。
- 计划实施后，需要同步更新 `docs/codex/PROJECT_CONTEXT.md`、`docs/product/InvestInsight-product-overview.md` 和版本文档。
