# 板块详情聚焦布局设计稿

最后更新：2026-06-26

## 目标

当前板块详情页保留了大量有价值信息，但信息层级过平，用户需要从很长的页面中自行寻找重点。新版布局目标是让用户打开详情后先回答四个问题：

1. 现在建议怎么做：增配、观察、减仓、过热不追或风险预警。
2. 什么时候适合看：观察买入区、止损失效位、止盈减仓区。
3. 为什么这么判断：趋势图、技术结构、事件影响路径。
4. 之后看什么：短中长期预测、关键验证事件、新闻和资金证据。

## 布局原则

- 首屏聚焦决策，不再把核心评分、AI 文本、事件、技术表格全部平铺。
- 趋势图必须保留，并放在首屏主体区域，承接点位计划。
- 购买建议、相关事件分析和预测必须在首屏或首屏下方第一屏内出现。
- 新闻证据、资金流明细、相关板块、回测和数据质量不删除，但移入“证据层”，默认以摘要或折叠组展示。
- 每个模块只回答一个问题，避免一个区块同时承担结论、解释、数据和风险。

## 页面结构

| 区域 | 目的 | 保留信息 |
| --- | --- | --- |
| 顶部状态条 | 告诉用户当前板块和运行状态 | 板块名、今日涨幅、数据日期、规则/AI 状态 |
| 决策摘要 | 先给出可执行建议 | 建议动作、置信度、方向分、入场时机、风险收益比 |
| 点位计划 | 说明何时观察或退出 | 观察买入区、止损失效位、止盈减仓区、持有周期 |
| 趋势图 | 让用户核对价格结构 | K 线、MA、支撑/阻力区、成交量、MACD |
| 事件与预测 | 解释未来为什么可能变好或变坏 | 事件状态、影响路径、短中长期预测、验证节点 |
| 证据层 | 保留完整信息但降低首屏噪音 | 新闻、资金、相关板块、回测、技术指标、数据质量 |

## 实现备注

- 后续实现时优先改 `src/ui/renderers/SectorDetailRenderer.cpp`，不要把逻辑塞回 `MainWindow.cpp`。
- `ChartRenderer` 需要支持在图上叠加观察区、止损线和止盈区；如果暂时不改图表绘制，可先在图表右侧用点位卡片表达。
- 证据层可先用 HTML 摘要块实现，不必一次做真实折叠交互；但视觉上需要体现“默认收束”。
- UI smoke 需要新增对“决策摘要、趋势图、事件预测、证据层”的文案验证。
